> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Add 4 missing admin tabs: Trust Policies, Cost Dashboard, Procedural Workflows, Agent Status

**Architecture:** Each tab is a self-contained Alpine.js component in admin_ui.py fetching from existing REST endpoints. No new backend code needed — all data already exposed by trust_engine.py, token_budget.py, procedural_memory.py, and agents/. CSS follows existing tab pattern.

**Tech Stack:** Alpine.js (existing), Chart.js (existing in admin_ui), existing FastAPI endpoints

## Codebase Context

### What already exists (do NOT rebuild)

All 4 tab sections are already defined as Python string constants in `src/bridge/admin_ui.py`:
- `SECTION_TRUST` (line 735) — table of policies + audit log
- `SECTION_COSTS` (line 782) — budget gauge + model breakdown table
- `SECTION_WORKFLOWS` (line 832) — workflow list with auto-suggest toggles
- `SECTION_AGENTS` (line 863) — agent cards + execution history table

All 4 are already referenced in the Alpine.js `loadTab()` switch (lines 1205–1208) and wired to their data-loading functions:
- `loadTrust()` → `GET /trust/policies` + `GET /trust/audit?limit=50`
- `loadCosts()` → `GET /budget/status` + `GET /budget/daily-report`
- `loadWorkflows()` → `GET /workflows`
- `loadAgents()` → `GET /agent/status` + `GET /agent/history`

All 4 tabs appear in the navigation list (lines 1121–1123).

### What is missing

**No tests exist** for any of these 4 tabs or their backing endpoints. The file `tests/test_admin_ui_tabs.py` does not exist. The tasks below are purely about writing tests and verifying the existing implementation is correct end-to-end.

## Known gaps vs spec (follow-up items, out of scope for this plan)

These items are specified in the spec but not yet implemented in the current codebase. They can be tracked as follow-up tasks:

1. **Cost Dashboard — 7-day bar chart**: `SECTION_COSTS` (lines 782–827) does not contain a `<canvas>` element for the Chart.js history chart. The spec calls for one using `GET /budget/history`. The endpoint exists (`/budget/history` in token_budget.py line 351) but the canvas + chart init JS are absent.

2. **Trust Policies — auto-promote threshold controls**: The spec calls for an "Auto-promotion controls" column with the `auto_promote_after` threshold. `SECTION_TRUST` renders `successful_executions` and `failed_executions` but not `auto_promote_after` or `last_promoted_at`.

3. **Trust Policies — Cancel button for pending audit entries**: The spec calls for a "Annuler" button on `pending_notify` outcomes using `POST /trust/cancel/{audit_id}`. The endpoint exists in trust_engine.py (line 467) but the HTML is not in `SECTION_TRUST`.

4. **Procedural Workflows — Delete button**: The spec calls for a "Supprimer" button using `DELETE /procedural/workflows/{id}`. No delete endpoint exists in extensions.py and no button is in `SECTION_WORKFLOWS`.

5. **Agent Status — cost per agent run**: The spec calls for cost per run. The `_agent_history` deque stores `tokens` but not a dollar cost. Would need a cost computation step in `agent_run_endpoint`.
